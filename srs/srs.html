1.Introduction

1.1 Purpose

The name of our software is document digitization service. Digital system is a system in which we can store, process or communicate any kind of information in a digital form. Day by day, we are entering a digitalized world, so all the management systems in every sector also need to be digitalized too. For the purpose of living an easier and better life , everything is being digitized. we need to make a system by which we can convert the offline records into the digitized version of record keeping. Then the records of the converted digital document will be stored in a database system of the finance department. Search and retrieval works can be done by the following system. Three different users will be there in the system such as public, private, administrators.

1.2 Document Conventions

In our following document, the format we used is IEEE SRS format. There the main topic will be in bold format and our font size is 30 for the topics, 18,20 will be for subsection of the topics. There will be 6 sections of the main topics with prefixes and the sub topics will be like 1.1, 1.2, 1.3. The special names and technical terminologies will be italicized.

1.3 Intended Audience and Reading Suggestions

There are different types of readers that the document is intended for, such as administrators, project managers, developers and the department of finance. There will be 6 sections here and each section will have subsections also. The first section is the introductory part. The second section is all about summary. Then comes the third section and it's all about the user, software, hardware and communication interfaces. In section four, there is summarization of the whole system features. All the non-functional requirements are in the fifth section and other requirements are given in section six.

1.4 Product Scope

This software will keep all the files or documents in an online secured place so that the official user or public can access the files publicly. Also the official users can get a private workspace there to upload their private files or retrieve their files. Another advantage is this is time consuming because users don’t have to exchange or get their files or documents through a person, they can easily get their things from this online secured place from anywhere at any time. Bangladesh is now getting digitized day by day. To digitize the whole country, we need to develop the root first. Such as the gov official working things like the departments of the finance sector. If we don’t digitize it then, the vision of digitization of Bangladesh will not work. The goal of this software is mainly keeping the whole offline system safely in this workplace, so that all the files and documents can be managed easily.

2. Overall Description

2.1 Product Perspective

The system was designed to keep records of documents in a digital way. In fact, this system will work to digitize the finance department of an organization. Their document was previously handwritten which will be digitized through this system. This system can be used to digitize the existing document search, add, delete documents. The system adds new users by administrative way and the users can manage their own documents. This software will consist of web application and mobile application. The web application will work for all the devices and will support in mozilla firefox, chrome, opera etc.The system uses the same database for web application and mobile application and that will integrate the overall user in a single cloud.

2.2 Product function

The users will be able to upload the files and documents through the mobile applications and web applications.The product has different types of functionality.The first functionality is taking input from the users in different types of documents such as PDF,PNG etc.Then the second functionality is to digitize the old data and process the data that will come in the future and keep it together. And keep it in such a way that it can be found by searching. And there will be public and private folders.

The basic things that can be done with this system are:

1. New users can be created.

2. The system will take the documents in its required format and the user will be able to upload as much data as possible.

3. There will be public and private folders. And the user can lock his folder if he wants.

4. User will be able to upload his avatar.

5. The files will be found by searching.

6.merge the mobile and the web data in a single cloud and that will be helpful for the future.

2.3 User classes and characteristics

We have different classes of users in this system. They will be limited to their specific tasks. For example, the administrator will work on how to keep the whole system in good condition and when new users come, they will register through a process. And the job of the users of the finance department is to digitize the documents and oversee them. And the ordinary people who are different will just be able to see the documents. And they will work here one by one as an actor and a class and they will perform their own work.

2.5 Design and Implementation Constraints

In our web application the main constraints are android version and internet connection. Without internet connection the user and the administrator won’t be able to access the application. The administrator can't control the application without internet connection. The user access will also be denied without internet connection. The android version less than 10.0 will also not be supported by the web application. All the documents will be saved in a database with security. Only the administrator will have the access over the whole database and everything. The public data will be shared or accessible for everyone. For security and privacy concerns, the particular data will be accessed by individual users.

3. External Interface Requirements

3.1 User Interfaces

The user interface is the main interface for the web application.

The first Image is for Login page and the second page is for entering username, then the last page is for homepage.

3.2 Hardware Interfaces

The device that we will connect to application and run, will need an internet connection with it. In case of a computer, we will need a scanner by which we will scan the documents and upload. For mobile devices, we will basically need a built in camera so that we can scan the documents and upload it. In that case, users will need the access to the camera from that application for using the camera to scan.

3.3 Software Interfaces

Native android development kit will be used to develop the app . The OCR library will be used to understand the characters . To make the app fast and stable, MYSQL database will be used to store the data and sync properly through APIs. A PDF renderer will be implemented that will let the user open a large file.different web browsers will be used. python and Django will be used .

3.4 Communications Interfaces

This application will be used for scanning the documents.and recognize the documents in a suitable format and upload it to the system. documents will be stored in a cloud and a single document will be max(5 MB) of size.and that could be happened by the HTTP response.

5. Other Non-Functional Requirements

5.1 Performance Requirements

There are some performance requirements like no page will take more than 2 seconds to access for one user. If there are any 3 failed login attempts, the account should be blocked. Then. the assigned administrator who uses the administration panel will allow you to reset the password. For no functional change in the system for maximum 15 minutes, the system will get automatically logged out. The system will have to have the advanced system of recognizing the optical character recognition (ocr) of the documents which are going to be scanned.

5.2 Safety Requirements

The system should be capable of Google Login or Active Directory as login authentication, so that, if the username is forgotten or any type of login problem for missing data or the device is damaged, then it will save data in google cloud.

5.3 Security Requirements

  1. The accounts have to have only alpha and numeric keys, that are allowed in the username and password field.
  2. The data of the database will never be transmitted to unauthenticated sources.
  3. The system will never reveal administrator information that is authorized to the users.
  4. At the rest, the database must be encrypted.

5.4 Software Quality Attributes

Adaptability:Users can understand the functions easily as the database is normalized.
Availability:The web applications will be available for 24/7 hours as the system will be hosted in cloud servers.
Correctness:The system will receive the appropriate information from the original documents and then will use Google’s API for providing accurate information.
Flexibility:There is flexibility to find the data by searching index, date, file name etc.
Reliability:Users can use dashboard to streamline authentication, asset protection and operational control.
Portability:The server will support in Linux, Windows, Mac and for Mobile devices : Android 10 or later.
Testability: It's a property that measures a piece of code or functionality to test. It's a testing of basic functions properly works or not like, creating folders, scanning and uploading documents etc.
Usability: Only the officials of the department of finance can login to the system.

5.5 Business Rules

  1. The users can login to the system only when he/she is a member of the department of finance and have completed the application form given by the Administrator.
  2. The administrator has the power of ensuring valid information and access to the information of employees including their shared or uploaded data. He also can remove the user id.
  3. The member who is given power can control all the employees information and their shared or uploaded documents.
  4. The member of the department of finance can login the system by using specific rules and can digitize data.
